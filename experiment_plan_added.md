（单包特征 + 历史行为特征）实验计划

本计划将新实验定义为 Group E：以"当前包"为预测对象，直接复用现有 Group A 的约 28 维单包特征和 Group B 的约 55-58 维时间窗口行为统计特征，并在历史不足时加入窗口完整度与密度类补偿特征。实验只对表征相关参数进行调优，主要包括采样步长 S 与历史窗口大小 W；模型参数保持为现有项目中的固定设置。

# 1\. 现有项目基础与新增实验定位

现有项目已经构建了多粒度 IoT 设备识别实验框架，包括：A 组单包级表征、B 组时间窗口级统计表征、C 组压缩序列表征、D 组原始序列表征。新增实验将作为 A 与 B 之间的融合组，引入"当前包 + 历史上下文"的混合粒度表示。

| **实验组** | **粒度**   | **特征维度** | **说明**                                       |
| ---------- | ---------- | ------------ | ---------------------------------------------- |
| A          | 单包级     | 约 28 维     | 当前包的包头/方向/协议等即时特征               |
| B          | 时间窗口级 | 约 55-58 维  | 固定时间窗口内的行为统计特征                   |
| C          | 压缩序列级 | 31 维        | 序列压缩后的统计表征                           |
| D          | 原始序列级 | 25 维 × H    | 原始时序输入 LSTM/GRU                          |
| E（新增）  | 混合粒度   | 约 90-92 维  | 当前包单包特征 + 历史行为统计 + 冷启动补偿特征 |

# 2\. 新实验组的目标与研究问题

- 验证"当前包即时属性"和"短时历史行为上下文"是否具有互补性。
- 比较不同表征参数设置下，混合粒度表征对设备识别性能的影响。
- 在不修改原有特征定义的前提下，确保新增实验与现有 A 组、B 组结果可直接对比。
- 分析采样步长与历史窗口大小对模型效果、样本数量和表征稳定性的影响。

# 3\. 样本构造规则（Group E）

每条 Group E 样本对应一个被选中的"当前包" p_t。最终特征由三部分拼接组成：

- 当前包单包特征：直接复用 Group A 的约 28 维特征，不做改动。
- 历史行为统计特征：以当前包前的固定历史窗口为统计范围，直接复用 Group B 的约 55-58 维统计特征定义，不做改动。
- 冷启动补偿特征：当历史窗口不足时，加入窗口完整度与密度相关特征，帮助模型区分"短历史统计"和"完整历史统计"。

## 3.1 当前包选择方式

采用固定时间采样步长 S 生成锚点，并为每个锚点选择一个当前包。具体规则如下：

- 设流量片段起始时间为 t0，结束时间为 tend，采样步长为 S。
- 锚点集合定义为：t0 + S, t0 + 2S, t0 + 3S, …，直至不超过 tend。
- 对于每个锚点 tk，选择时间戳不早于 tk 的第一个到达包作为当前包。
- 尾部若不足一个完整采样步长，则直接丢弃，不再额外补尾部锚点。

## 3.2 历史窗口定义

对每个当前包，仅使用它之前的历史流量计算行为统计，禁止使用当前包本身及其未来流量，以避免信息泄露。

- 历史窗口采用时间定义，而不是包数定义。
- 若当前包时间为 t，则历史集合为 (t−W, t) 内的所有包。
- 历史窗口大小 W 为待调表征参数。
- 窗口内所有行为统计特征的计算方法保持与现有 Group B 一致，仅将统计对象改为"当前包之前"的历史流量。

## 3.3 冷启动处理策略（保留）

对于流量开头的样本，可能不存在完整的 W 秒历史。此类样本不丢弃，而是保留已有历史并加入补偿特征。

| **特征类别** | **变量名称（示例）**     | **作用**                        |
| ------------ | ------------------------ | ------------------------------- |
| 完整度       | hist_duration_actual     | 记录当前样本真实可用的历史时长  |
| 完整度       | hist_window_ratio        | 记录实际历史时长 / 目标窗口时长 |
| 完整度       | hist_complete_flag       | 标识该样本是否具备完整历史窗口  |
| 样本规模     | hist_packet_count_actual | 记录用于统计的历史包数          |
| 密度         | packet_rate              | 历史包数 ÷ 实际历史时长         |
| 密度         | byte_rate                | 历史字节数 ÷ 实际历史时长       |

这些补偿特征不是为了"补齐"前期样本，而是显式告诉模型：该统计量是在多长历史、多大样本规模下得到的，从而缓解前期和后期样本统计基础不同的问题。

# 4\. 数据处理与切分规范

- 继续使用现有项目数据集与时间顺序划分方案：训练集 70%，验证集 15%，测试集 15%。
- 必须先按时间切分原始 packet-level 数据，再分别在训练集、验证集、测试集内部独立构造 Group E 样本。
- 禁止在全数据集上先构造历史特征再切分，否则测试样本的历史可能引用到训练或验证阶段的数据，造成信息泄露。
- 尾部不足一个采样步长的残余片段直接丢弃；开头历史不足窗口的样本保留并增加补偿特征。

# 5\. 特征使用原则

本实验不重新设计原有特征，不修改 A 组和 B 组既有定义。这样做的原因是：

- 保持与前面实验的可比性，确保新增实验的性能变化主要来自"融合表征"而不是"特征重写"。
- 避免引入额外变量，使后续结果解释更清晰：提升可归因于"当前包 + 历史行为"的联合建模。
- 现有约 28 维单包特征与约 55-58 维行为特征拼接后，总维度约 90-92 维，对 RF / XGBoost 来说并不算高，不构成必须重设计特征的理由。

# 6\. 表征参数调优方案

本实验只调优样本构造相关参数，不调模型超参数。模型训练统一使用现有项目中的固定默认设置。调优顺序如下：

- 第一步：固定模型配置，仅比较不同采样步长 S 与历史窗口大小 W 的组合。
- 第二步：在验证集上以 Macro-F1 作为主指标选出最佳表征组合。
- 第三步：使用最佳表征组合在测试集上进行最终评估，并与 A 组、B 组结果比较。

## 6.1 待调参数

| **参数**     | **符号** | **候选值**    | **说明**                   |
| ------------ | -------- | ------------- | -------------------------- |
| 采样步长     | S        | 5s, 10s, 20s  | 控制当前包锚点的时间间隔   |
| 历史窗口大小 | W        | 10s, 30s, 60s | 控制用于行为统计的历史时长 |

首轮参数搜索共 9 组表征配置：3 个采样步长 × 3 个历史窗口。若运行成本较高，可先缩减为 S={10s,20s}, W={10s,30s,60s} 的 6 组实验。

## 6.2 调优原则

- 主调优指标使用验证集 Macro-F1，原因是其更能反映不同设备类别之间的平衡识别效果。
- 当多个组合的 Macro-F1 非常接近时，再参考 Weighted-F1 和 Accuracy 进行辅助判断。
- 若某一组合效果略高，但样本量明显过少或计算代价显著上升，应同时记录其工程可用性，避免仅凭单点精度选择。

## 6.3 推荐的调优流程

步骤 1：基于已有 packet_level.csv，按照训练 / 验证 / 测试的时间顺序切分数据。

步骤 2：对每个候选 S、W 组合，分别在训练集、验证集、测试集内部独立生成 Group E 样本表。

步骤 3：使用固定模型设置在训练集训练，在验证集评估并保存指标。

步骤 4：根据验证集 Macro-F1 选择最佳 S、W 组合。

步骤 5：在最佳表征下重新训练并在测试集报告最终指标。

步骤 6：将 Group E 的最终结果与现有 A 组、B 组结果统一汇总。
