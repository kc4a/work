**原始 packet-level CSV 字段表**

内容重编排版（不改变原文含义）

# 一、样本索引与标签字段

这些字段主要用于数据组织、排序、划分和打标签，不直接输入模型。

| **字段名** | **含义** | **用途**                                      |
| ---------- | -------- | --------------------------------------------- |
| ts         | 时间戳   | 用于时间排序、窗口切分、计算间隔和 burst 特征 |
| device_id  | 设备标签 | 监督学习标签                                  |

# 二、地址与基础通信字段

| **字段名** | **含义**                 | **用途**                                         |
| ---------- | ------------------------ | ------------------------------------------------ |
| src_ip     | 源 IP 地址               | 用于数据清洗、方向判断、标签映射；不直接用于训练 |
| dst_ip     | 目的 IP 地址             | 用于数据清洗、方向判断、标签映射；不直接用于训练 |
| src_mac    | 源 MAC 地址              | 可选保留，用于标签映射、调试；不直接用于训练     |
| dst_mac    | 目的 MAC 地址            | 可选保留，用于标签映射、调试；不直接用于训练     |
| src_port   | 源端口                   | 可直接用于训练，也可用于后续统计特征构造         |
| dst_port   | 目的端口                 | 可直接用于训练，也可用于后续统计特征构造         |
| direction  | 方向（inbound/outbound） | 可直接用于训练，也是窗口和序列分析的重要特征     |

# 三、包大小与负载特征

这部分是 packet-level 的核心低层行为特征。

| **字段名**      | **含义**         | **用途**                       |
| --------------- | ---------------- | ------------------------------ |
| packet_len      | 整个包的长度     | 单包、窗口、序列实验的核心特征 |
| payload_len     | 有效载荷长度     | 反映实际业务内容大小           |
| has_payload     | 是否存在有效载荷 | 区分控制包、空包、数据包       |
| payload_entropy | 载荷熵           | 描述载荷内容随机性             |

# 四、时间特征

| **字段名** | **含义**                   | **用途**                                       |
| ---------- | -------------------------- | ---------------------------------------------- |
| delta_t    | 当前包与前一个包的时间间隔 | 单包实验中的局部节奏特征，也用于窗口与序列统计 |

# 五、协议与链路层特征

| **字段名** | **含义**                        | **用途**           |
| ---------- | ------------------------------- | ------------------ |
| eth_type   | 链路层协议类型                  | 描述底层协议环境   |
| ip_version | IP 版本（IPv4/IPv6）            | 协议栈实现相关特征 |
| ip_proto   | IP 上层协议类型（TCP/UDP/ICMP） | 描述协议偏好       |

# 六、IP 层结构特征

这部分不是地址，而是 IP 层行为/结构特征，可以用于训练。

| **字段名** | **含义**            | **用途**                       |
| ---------- | ------------------- | ------------------------------ |
| ip_len     | IP 层总长度         | 比 packet_len 更贴近网络层结构 |
| ip_ttl     | TTL（生存时间）     | 反映设备协议栈差异             |
| ip_tos     | TOS / DSCP          | 反映服务类型与 QoS 偏好        |
| ip_flags   | IP 标志位           | 反映分片与协议栈行为           |
| ip_ihl     | IP 头长度           | 描述 IP 头部结构               |
| ip_df      | Don't Fragment 标志 | 描述分片策略                   |

# 七、TCP 层特征

这部分对设备识别通常很重要。

| **字段名**  | **含义**     | **用途**                        |
| ----------- | ------------ | ------------------------------- |
| tcp_syn     | SYN 标志     | 反映连接建立行为                |
| tcp_ack     | ACK 标志     | 反映确认行为                    |
| tcp_fin     | FIN 标志     | 反映连接结束行为                |
| tcp_rst     | RST 标志     | 反映异常终止或拒绝行为          |
| tcp_psh     | PSH 标志     | 反映数据推送行为                |
| tcp_urg     | URG 标志     | 补充少见控制行为                |
| tcp_window  | TCP 窗口大小 | 反映 TCP 栈实现和通信策略       |
| tcp_dataofs | TCP 头长度   | 描述 TCP 头部结构与选项使用情况 |

# 八、UDP / ICMP 特征

| **字段名** | **含义**     | **用途**           |
| ---------- | ------------ | ------------------ |
| udp_len    | UDP 长度字段 | 补充 UDP 通信模式  |
| icmp_type  | ICMP 类型    | 描述 ICMP 报文语义 |
| icmp_code  | ICMP 代码    | 补充 ICMP 报文语义 |

# 九、可选辅助派生字段

这些字段不是必须，但如果提取方便，后续构造窗口特征会更方便。

| **字段名**         | **含义**             | **用途**                    |
| ------------------ | -------------------- | --------------------------- |
| dport_class        | 目的端口类别         | 用于端口类别统计            |
| is_https           | 是否访问 80/443 端口 | 用于窗口级 https_port_ratio |
| is_well_known_port | 是否属于常见服务端口 | 用于窗口级端口行为统计      |
