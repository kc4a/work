# YourThings 家族级跨数据集验证报告

## 1. 实验目的

本轮实验使用 `YourThings` 数据集，对当前项目做一组新的跨数据集验证。

与此前基于 `UNSW-IoTraffic` 的设备级跨时间验证不同，这一轮采用的是**家族级标签映射**：

- 不再要求测试集设备与训练集设备一一精确对应；
- 而是将设备映射到更粗粒度的功能家族，如 `camera`、`speaker`、`hub`；
- 用来回答一个更核心的问题：模型跨域后是否仍保留一定的“行为家族识别能力”。

因此，这组实验的定位应当是：

- 强异构外部家庭网络验证；
- 家族级跨数据集验证；
- 对设备级跨域结果的补充，而不是替代。

## 2. 数据构造

### 2.1 数据来源

- 训练集：`data/16-09-23.pcap` 解析得到的原始数据集
- 测试集：`data/2018/03/20` 下的 `YourThings` 连续 5 分钟切片
- 设备映射文件：`device_mapping.csv`

### 2.2 家族级映射策略

训练集和测试集统一映射到以下家族标签：

- `camera`
- `speaker`
- `hub`
- `motion_sensor`
- `plug_switch`
- `baby_monitor`
- `light`

但真正同时出现在训练集与测试集中的共同家族只有 6 类：

- `baby_monitor`
- `camera`
- `hub`
- `motion_sensor`
- `plug_switch`
- `speaker`

`light` 虽然在 YourThings 中可见，但训练侧映射样本不足以进入本轮最终共同标签集合，因此未纳入正式评估。

### 2.3 测试集抽取方式

为了避免直接处理完整日级流量造成计算成本过高，本实验采用按家族限额的连续截取策略：

- 顺序扫描 `2018-03-20` 当天全部 288 个连续切片；
- 每个家族最多保留前 `50,000` 个已识别包；
- 最终生成测试 CSV：
  - `data/processed/yourthings_family_packet_level.csv`

## 4. 结果解读

### 4.1 哪些方案最好

在 YourThings 这种更强异构的外部家庭网络上，表现最好的方案是：

- `D-LSTM`：`Accuracy = 0.3695`，`Macro-F1 = 0.2957`
- `E1-RF`：`Accuracy = 0.3495`，`Macro-F1 = 0.2175`

这说明在强外部域偏移下，仍然最有价值的是两类信息：

- 原始序列中的时序行为模式；
- 当前包相对历史基线的关系特征。

### 4.2 哪些方案明显失效

最差的是 `B-RF`，`Accuracy` 只有 `0.0414`。

这说明单纯依赖固定窗口统计，在 `YourThings` 这种不同家庭环境、不同设备组合、不同网络拓扑下几乎无法保留稳定判别能力。

`A` 组同样较弱，说明单包外观特征跨到强异构新域后也大幅失效。

### 4.3 家族级可迁移性是否存在

存在，但非常有限。

从代表性模型看：

- `D-LSTM` 中 `camera` 的 `F1 = 0.5849`，`plug_switch` 的 `F1 = 0.4484`，`motion_sensor` 的 `F1 = 0.3721`
- `E1-RF` 中 `speaker` 的 `F1 = 0.5334`，`motion_sensor` 的 `F1 = 0.4558`

这说明跨域后并不是完全随机，而是仍然保留了少量“行为家族”的可迁移结构。

但同时也能看到：

- `hub` 在 `D-LSTM` 中 `F1` 接近 0
- `baby_monitor` 在 `E1-RF` 中完全失效

所以更准确的结论不是“家族级泛化已经成立”，而是“只保留了有限的粗粒度迁移性”。

## 5. 与前面跨数据集实验的关系

这轮 `YourThings` 结果与 `UNSW-IoTraffic` 的设备级结果有明显不同：

- `UNSW-IoTraffic` 更像同设备跨时间、跨采集批次验证，因此 `B-RF`、`E1-RF` 仍能保持较高分数；
- `YourThings` 是更异构的外部家庭网络，域差异更强，因此即使降到家族级，整体分数也明显更低。

因此两轮实验组合起来，可以形成更完整的跨域结论：

- 在中等强度跨域下，窗口级与关系增强方法仍然稳定；
- 在强异构外部域下，设备级特征大幅失效，只剩少量家族级可迁移性；
- 这说明当前模型尚未学到足够稳健的“设备本质表征”。

## 6. 结论

本轮 YourThings 家族级跨数据集验证表明：

1. 将标签放宽到功能家族后，模型确实还能保留一部分跨域识别能力。
2. 这种能力主要体现在 `camera`、`speaker`、`plug_switch`、`motion_sensor` 等少数家族上。
3. 但总体 `Macro-F1` 仍然不高，说明跨域可迁移性非常有限。
4. `D-LSTM` 和 `E1-RF` 相对更强，说明时序模式和关系特征比静态单包/窗口统计更有外部迁移潜力。
5. 因此，YourThings 的结论应被表述为：当前方法证明了“有限的家族级可迁移性”，但尚不能证明具备稳健的设备级跨域泛化能力。
